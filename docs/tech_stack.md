Flutter layer — Flutter + Flame is non-negotiable for you. Flame handles all particle effects, territory polygon animations, and screen shake on capture. Riverpod gives you type-safe reactive state which you'll need when GPS, WebSocket, and game state all need to sync simultaneously. flutter_map with OpenStreetMap avoids Google Maps billing surprises as you scale to 500K+ users.
Backend core — Socket.IO is your most critical dependency. Territory capture notifications, clan war updates, and live leaderboards all run through it. Redis isn't just a cache — your entire leaderboard system (sorted sets) lives in Redis because it handles real-time ranked queries in milliseconds that MongoDB cannot.
Geo layer — PostGIS + Turf.js is where most teams underestimate complexity. Turf.js runs the polygon intersection math on the Node.js side (buffer, simplify, difference, intersect), while PostGIS handles the $geoIntersects queries efficiently at scale. You need both — Turf for calculations, PostGIS for querying.
AI/ML — TensorFlow Lite runs on-device for form correction and meal photo recognition (no server round-trip needed for basic inference). PyTorch on the server handles heavier personalized coaching models. Keep them separate — on-device for speed, server-side for accuracy.
One critical addition not in your doc — for your Phase 1 India launch, add Razorpay for payments instead of Stripe. Razorpay handles UPI, Paytm, and Indian credit cards natively, which is essential for converting your urban India user base to paid subscribers.
